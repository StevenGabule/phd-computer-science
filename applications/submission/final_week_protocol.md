# Final-Week Submission Protocol

**Purpose:** Day-by-day protocol for the week leading into each major deadline cluster (Dec 1, Dec 8, Dec 15). The goal is to make submission day boring.

---

## Pre-week (Sunday before deadline week)

- [ ] Read every active row in `deadline_tracker_template.csv` for programs due this week
- [ ] Read every active row in `blockers.md`
- [ ] Verify your CV, SOP, and PHS files are at their current final versions (commit to git: `git tag -a app-v[date] -m "application materials snapshot"`)
- [ ] Verify all letters expected this week have been received (check each portal)
- [ ] Verify all transcripts expected this week have been received
- [ ] Verify fee waivers are approved (or budget for unpaid fees)
- [ ] Verify your application portals are all set up with completed fixed fields (name, transcripts, CV, demographics)
- [ ] If there's >5 unfinished items on this list, you're behind. Reschedule lower-priority tasks for this week.

## Monday — Internal review of each application's per-program SOP

- [ ] Open each program's per-program SOP customization
- [ ] Read aloud (catches typos and awkward phrasing better than silent reading)
- [ ] Verify advisor mentions are current (recheck Google Scholar for any 2027 papers that should replace the cited 2024-2026 ones)
- [ ] Verify lab / center names haven't changed since the SOP was written
- [ ] Confirm SOP word count is under the program's hard limit (use `wc -w` on the markdown)
- [ ] Save final PDFs of each per-program SOP

## Tuesday — Buffer day for blockers

- [ ] Address any active blockers from `blockers.md`
- [ ] If a recommender hasn't sent a T-7 acknowledgment for a Friday deadline, send the T-7 reminder
- [ ] Test PDF uploads on each portal (some portals reject files larger than 5MB; compress now if needed)
- [ ] Test that you can log into each portal (in case password reset is required)

## Wednesday — Submit early-window applications

- [ ] Submit Dec 1 / Dec 8 deadline applications (whichever is in scope this week)
- [ ] Submission protocol per application (~15-30 min each):
  1. Log in to portal
  2. Upload SOP PDF
  3. Upload PHS PDF (if required)
  4. Upload CV PDF
  5. Upload research proposal (Europe programs only)
  6. Upload writing sample (CMU LTI only)
  7. Upload transcripts (or verify already received)
  8. Verify recommender list is complete (3 names + emails)
  9. Verify fee waiver applied OR enter payment card
  10. Final review pass: read your application as you'd read someone else's
  11. Click Submit
  12. Take screenshot of confirmation page
  13. Verify confirmation email received within 30 minutes
- [ ] Update `deadline_tracker_template.csv` with submission status + timestamp
- [ ] If submission fails for ANY reason, log in `blockers.md` immediately and contact program admissions

## Thursday — Buffer day + edge-case handling

- [ ] Address any portal issues from Wednesday's submission run
- [ ] If a recommender T-7 hasn't been acknowledged, send a follow-up T-2 (see followup_emails/T-2.md)
- [ ] If a transcript hasn't arrived, contact the source registrar
- [ ] Begin Friday deadline submissions (Dec 8 / Dec 15 cluster) — submit ahead of the actual deadline

## Friday — Final submissions + verification

- [ ] Submit any remaining applications for this week
- [ ] Verify EVERY application submitted this week shows "Complete" status (all documents received, all letters in)
- [ ] If any application is incomplete, address before end of day
- [ ] Confirmation log in `journal/decisions.md` for the week's submissions

## Saturday — Rest day

- [ ] Do not open applicant portals
- [ ] Do not refresh recommender status pages
- [ ] You've earned this

## Sunday — Plan for next week's deadlines

- [ ] Repeat the pre-week protocol for next week's deadlines

---

## What to do if you're behind on Monday

If on Monday morning more than 5 items from the pre-week list are unchecked:

- **Triage by deadline:** focus on programs with the earliest deadline first (Dec 1 > Dec 8 > Dec 15)
- **Triage by tier:** within a deadline cluster, focus on reach programs first (these have the lowest acceptance rates; least margin for incomplete materials)
- **Defer match programs to Tuesday/Wednesday:** match programs are more forgiving of minor application weaknesses
- **Drop safety programs only if absolutely required:** these are your fallback acceptances; cutting them is a serious decision

---

## What to do on submission day if everything is going wrong

1. **DO NOT panic-submit.** A rushed application is worse than a 12-hour-late application.
2. Email program admissions FIRST, then debug. Most programs grant a 24-72 hour grace period if you flag a technical issue before the deadline.
3. Document everything in `blockers.md` as it happens — screenshots, error messages, the steps you've tried.
4. If you genuinely cannot submit and admissions does not respond, write a 1-paragraph note in the portal's "additional information" field (if available) explaining the issue and click Submit anyway.

---

## Mid-deadline-week checkpoint

Every Wednesday evening:
- [ ] Count: submitted this week / target for this week
- [ ] If below target by Wednesday, schedule Thursday morning as a "no-meetings, all submissions" block
- [ ] If a recommender has been silent on multiple programs, send a personal note (not the templated reminders) asking if everything is OK

---

## Final-week sleep protocol

- Sleep 8 hours each night during deadline weeks. Tired errors at submission cost more than the time saved by staying up.
- Caffeine intake: stable. Spiking is worse than maintaining.
- One social commitment per week (dinner with friends, etc.). Isolation makes the crunch worse, not better.

This sounds preachy. It's also the difference between submitting clean applications and submitting frantic ones.
