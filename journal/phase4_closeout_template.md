## Usage notes

- **Be honest in the retrospective.** The point isn't to celebrate; it's to learn. If something went badly, name it specifically.
- **Quantify where possible.** Real numbers ($, hours, # applications) are more useful than impressions in 5 years.
- **Don't write this on Dec 31.** Write the Phase 4 section in early Jan 2028 after the deadline dust settles; write the 18-month closeout in early Feb after at least some interview / decision outcomes are visible.
- **Commit this to git.** Same as every other journal entry. The retrospective is part of the historical record.
- **Share selectively.** Some of this is personal; some is potentially useful to other applicants. The user decides what's public vs. private.
